# 外设真实化实现总结

## 完成的改造

### 1. Terminal (终端外设)
**文件**: `src/periph/Terminal.cpp`, `include/periph/Terminal.h`

**实现功能**:
- ✅ **INPUT 操作**: 使用跨平台宏 `GETCH()` 实现真实的终端字符输入
  - Windows: 使用 `_getch()` (conio.h)
  - Linux: 使用 termios 实现无回显输入
- ✅ **OUTPUT 操作**: 使用跨平台颜色宏实现带颜色的终端输出
  - Windows: `TERM_SET_COLOR(TERM_COLOR_RED)` 等
  - Linux: ANSI 转义序列 `\033[31;1m` 等

**关键代码**:
```cpp
void Terminal::handle_input(uint64_t vm_id) {
    char ch = GETCH();  // 跨平台无回显输入
    int result = static_cast<int>(ch);
    send_result(vm_id, result, false);
}
```

### 2. Disk (磁盘外设)
**文件**: `src/periph/Disk.cpp`, `include/periph/Disk.h`

**实现功能**:
- ✅ **READ 操作**: 从真实文件读取数据
  - 使用 `FILE_EXISTS()` 检查文件是否存在
  - 自动创建默认 4KB 磁盘镜像
  - 返回实际文件大小
- ✅ **WRITE 操作**: 写入数据到文件
  - 追加模式写入磁盘镜像文件
  - 使用跨平台路径分隔符 `PATH_SEP`

**关键代码**:
```cpp
void Disk::handle_read(uint64_t vm_id) {
    const char* filename = "disk_image.bin";
    if (!FILE_EXISTS(filename)) {
        create_default_disk_image(filename);
    }
    std::ifstream file(filename, std::ios::binary | std::ios::ate);
    std::streamsize size = file.tellg();
    send_result(vm_id, static_cast<int>(size), false);
}
```

---

### 3. NIC (网络接口卡)
**文件**: `src/periph/NIC.cpp`, `include/periph/NIC.h`

**实现功能**:
- ✅ **Socket 初始化**: 使用跨平台宏 `NET_INIT()` 
  - Windows: `WSAStartup(MAKEWORD(2, 2), &wsa)`
  - Linux: 无需初始化
- ✅ **RECV 操作**: 从 UDP socket 接收数据
  - 绑定到端口 9000
  - 非阻塞模式接收
  - 使用 `recvfrom()` 接收数据包
- ✅ **SEND 操作**: 发送数据到网络
  - 发送到 localhost:9001
  - 使用 `sendto()` 发送 UDP 数据包
- ✅ **Socket 清理**: 使用 `SOCKET_CLOSE()` 和 `NET_CLEANUP()`

**关键代码**:
```cpp
NIC::NIC(PeriphManager* pm) : socket_fd(INVALID_SOCKET_FD) {
    NET_INIT();  // 跨平台网络初始化
    socket_fd = socket(AF_INET, SOCK_DGRAM, IPPROTO_UDP);
    bind(socket_fd, (struct sockaddr*)&local_addr, sizeof(local_addr));
}
```

## 跨平台宏使用汇总

### 网络模块
```cpp
NET_INIT()           // 网络库初始化 (Windows: WSAStartup, Linux: 无操作)
NET_CLEANUP()        // 网络库清理 (Windows: WSACleanup, Linux: 无操作)
SOCKET_CLOSE         // Socket 关闭 (Windows: closesocket, Linux: close)
SOCKET_ERRNO         // 获取错误码 (Windows: WSAGetLastError, Linux: errno)
SocketType           // Socket 类型 (Windows: SOCKET, Linux: int)
INVALID_SOCKET_FD    // 无效 Socket (Windows: INVALID_SOCKET, Linux: -1)
```

### 文件模块
```cpp
MKDIR(path)          // 创建目录 (Windows: _mkdir, Linux: mkdir with mode)
FILE_EXISTS(path)    // 文件检查 (Windows: _access, Linux: access)
PATH_SEP             // 路径分隔符 (Windows: "\\", Linux: "/")
```

### 终端模块
```cpp
TERM_CLEAR()         // 清屏 (Windows: Console API, Linux: ANSI escape)
TERM_SET_COLOR(c)    // 设置颜色 (Windows: SetConsoleTextAttribute, Linux: ANSI)
TERM_COLOR_RED       // 红色 (Windows: FOREGROUND_RED, Linux: "\033[31;1m")
TERM_COLOR_GREEN     // 绿色
TERM_COLOR_RESET     // 重置颜色
GETCH()              // 无回显输入 (Windows: _getch, Linux: termios)
```

## 架构优势

1. **真实硬件交互**: 不再是模拟返回值，而是真正与文件系统、网络设备、终端交互
2. **跨平台兼容**: 同一套代码在 Windows 和 Linux 上都能正常工作
3. **可扩展性**: 基于消息路由架构，可以轻松添加更多真实外设
4. **调试友好**: 所有操作都有详细日志，便于排查问题

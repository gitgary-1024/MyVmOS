# 彩色日志系统实现文档

## 概述

参考 Python logging 库的设计，实现了一个功能完整的跨平台彩色日志系统，支持：
- ✅ 5 个日志等级（DEBUG, INFO, WARNING, ERROR, CRITICAL）
- ✅ 终端颜色输出（Windows/Linux）
- ✅ 日志等级过滤
- ✅ 线程安全
- ✅ 文件输出
- ✅ 自动解决 Windows GDI ERROR 宏冲突

---

## 核心特性

### 1. 日志等级（类似Python logging）

```cpp
enum class LogLevel {
    DEBUG = 10,      // 青色 - 详细调试信息
    INFO = 20,       // 绿色 - 一般信息
    WARNING = 30,    // 黄色 - 警告信息
    ERR = 40,        // 红色 - 错误信息 (使用 ERR 避免与 Windows GDI 宏冲突)
    CRITICAL = 50,   // 紫色 - 严重错误
    NOTSET = 0       // 未设置
};
```

### 2. 跨平台颜色支持

**Windows 平台**:
```cpp
#include <windows.h>
SetConsoleTextAttribute(hConsole, FOREGROUND_RED | FOREGROUND_GREEN);  // 黄色
```

**Linux/Mac平台**:
```cpp
#define COLOR_WARNING "\033[33m"  // ANSI 转义序列
std::cout << COLOR_WARNING << "Warning!" << COLOR_RESET;
```

### 3. 自动检测并解决宏冲突

Windows GDI 头文件 `wingdi.h` 中定义了：
```cpp
#define ERROR 0
```

**解决方案**:
```cpp
#ifdef _WIN32
    #ifdef ERROR
        #define WINDOWS_ERROR_MACRO_WAS_DEFINED
        #pragma push_macro("ERROR")  // 保存宏
        #undef ERROR                  // 临时取消定义
    #endif
#endif

// ... 使用 LogLevel::ERR ...

#ifdef _WIN32
    #ifdef WINDOWS_ERROR_MACRO_WAS_DEFINED
        #pragma pop_macro("ERROR")   // 恢复宏
    #endif
#endif
```

## 使用方法

### 基础用法

```cpp
#include "log/Logging.h"

int main() {
    auto& logger = Logger::instance();
    logger.set_level(LogLevel::DEBUG);
    
    LOG_DEBUG("Debug message");
    LOG_INFO("Info message");
    LOG_WARNING("Warning message");
    LOG_ERR("Error message");      // 注意：使用 LOG_ERR 而非 LOG_ERROR
    LOG_CRITICAL("Critical message");
    
    return 0;
}
```

### 带模块名的日志

```cpp
LOG_INFO_MOD("Router", "Router initialized");
LOG_DEBUG_MOD("VM", "VM created with ID=1");
LOG_WARNING_MOD("Disk", "Disk space low");
LOG_ERR_MOD("NIC", "Network connection failed");
LOG_CRITICAL_MOD("System", "System failure");
```

### 动态修改日志等级

```cpp
auto& logger = Logger::instance();

// 只显示 WARNING 及以上等级
logger.set_level(LogLevel::WARNING);

LOG_DEBUG("Hidden");     // 不显示
LOG_INFO("Hidden");      // 不显示
LOG_WARNING("Visible");  // 显示
LOG_ERR("Visible");      // 显示
```

### 文件输出

```cpp
auto& logger = Logger::instance();
logger.set_file_output("app.log");  // 输出到文件（无颜色）

// 或者输出到自定义流
std::ofstream my_stream("custom.log");
logger.set_output(my_stream);
```

## 输出格式

### 标准格式
```
[时间戳] [日志等级 ] [模块名] 消息内容
```

### 示例输出
```

[2026-03-08 20:04:33.657] [    INFO] [Router] Router initialized
[2026-03-08 20:04:33.657] [ WARNING] [Disk] Disk space low
[2026-03-08 20:04:33.657] [   ERROR] [NIC] Network connection failed
[2026-03-08 20:04:33.658] [CRITICAL] [System] Critical system failure
```

### 颜色对应关系

| 日志等级 | Windows 颜色 | Linux ANSI 代码 | 说明 |
|---------|-------------|----------------|------|
| DEBUG | 青色 (FOREGROUND_BLUE | FOREGROUND_GREEN) | `\033[36m` | 调试信息 |
| INFO | 绿色 (FOREGROUND_GREEN) | `\033[32m` | 一般信息 |
| WARNING | 黄色 (FOREGROUND_RED | FOREGROUND_GREEN) | `\033[33m` | 警告 |
| ERROR | 红色 (FOREGROUND_RED) | `\033[31m` | 错误 |
| CRITICAL | 紫色 (FOREGROUND_RED | FOREGROUND_BLUE) | `\033[35m` | 严重错误 |

## API 参考

### Logger 类方法

| 方法 | 说明 |
|------|------|
| `static Logger& instance()` | 获取单例实例 |
| `void set_level(LogLevel level)` | 设置日志等级 |
| `LogLevel get_level() const` | 获取当前日志等级 |
| `void set_output(std::ostream& os)` | 设置输出流 |
| `void set_file_output(const std::string& filename)` | 设置文件输出 |
| `void debug/info/warning/err/critical(msg, module)` | 日志记录方法 |

### 全局宏

| 宏 | 等价调用 |
|----|----------|
| `LOG_DEBUG(msg)` | `Logger::instance().debug(msg, __FILE_NAME__)` |
| `LOG_INFO(msg)` | `Logger::instance().info(msg, __FILE_NAME__)` |
| `LOG_WARNING(msg)` | `Logger::instance().warning(msg, __FILE_NAME__)` |
| `LOG_ERR(msg)` | `Logger::instance().err(msg, __FILE_NAME__)` |
| `LOG_CRITICAL(msg)` | `Logger::instance().critical(msg, __FILE_NAME__)` |

## 最佳实践

1. **选择合适的日志等级**
   - 开发阶段：`DEBUG`
   - 生产环境：`WARNING` 或 `ERROR`

2. **使用模块名区分来源**
   ```cpp
   LOG_INFO_MOD("MyModule", "Custom message");
   ```

3. **避免在性能敏感代码中使用 DEBUG**
   ```cpp
   // 好的做法
   if (condition) {
       LOG_DEBUG("Expensive debug info");
   }
   ```

4. **错误处理使用合适的等级**
   ```cpp
   try {
       // ... code ...
   } catch (const std::exception& e) {
       LOG_ERR(std::string("Exception: ") + e.what());
   }
   ```

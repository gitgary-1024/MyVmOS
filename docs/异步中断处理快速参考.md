# 异步中断处理快速参考

## 🚀 快速开始

### 1. 发送中断请求（VM 端）


```cpp

enum class InterruptPriority {
    LOW = 0,      // 低优先级
    NORMAL = 1,   // 普通优先级
    HIGH = 2,     // 高优先级
    REALTIME = 3  // 实时优先级
```

### InterruptType 枚举

```cpp
enum class InterruptType {
    TERMINAL_INPUT = 0,    // 终端输入 (NORMAL)
    TERMINAL_OUTPUT = 1,   // 终端输出 (NORMAL)
    DISK_READ = 2,         // 磁盘读 (LOW)
    DISK_WRITE = 3,        // 磁盘写 (LOW)
    NETWORK_RECV = 4,      // 网络接收 (LOW)
    NETWORK_SEND = 5,      // 网络发送 (LOW)
    TIMER = 6,             // 定时器 (NORMAL)
    KEYBOARD = 7,          // 键盘 (HIGH)
    MOUSE = 8,             // 鼠标 (HIGH)
    SYSTEM = 9             // 系统事件 (REALTIME)
};
```

### get_interrupt_priority() 函数

```cpp
// 自动获取中断类型对应的优先级
InterruptPriority get_interrupt_priority(InterruptType type);

// 示例
auto prio = get_interrupt_priority(InterruptType::KEYBOARD);  // 返回 HIGH(2)
```

### InterruptRequest 结构

```cpp
struct InterruptRequest {
    uint64_t vm_id;                          // VM ID
    int periph_id;                           // 外设 ID
    InterruptType interrupt_type;            // 中断类型
    InterruptPriority priority;              // 优先级（自动设置）

```

### InterruptResult 结构

```cpp
struct InterruptResult {
    uint64_t vm_id;                          // VM ID
    int return_value;                        // 返回值
```

## ⚡ 性能提示

### 1. 优先级选择建议

- **实时任务**（系统关键）→ 使用 `SYSTEM` 类型 → `REALTIME` 优先级
- **交互任务**（键盘鼠标）→ 使用 `KEYBOARD/MOUSE` 类型 → `HIGH` 优先级
- **普通任务**（终端定时）→ 使用 `TERMINAL/TIMER` 类型 → `NORMAL` 优先级
- **后台任务**（磁盘网络）→ 使用 `DISK/NETWORK` 类型 → `LOW` 优先级

### 2. 超时设置建议

```cpp
// 快速设备（键盘、鼠标）
timeout_ms = 500;  // 500ms

// 普通设备（终端、定时器）
timeout_ms = 2000;  // 2s

// 慢速设备（磁盘、网络）
timeout_ms = 5000;  // 5s
```

### 3. 性能监控

```cpp
// 推荐：每秒检查一次统计
while (running) {
    auto stats = VmManager::instance().get_interrupt_stats();
    
    if (stats.avg_latency_ms > 1000) {  // 平均延迟 > 1s
        std::cerr << "Warning: High interrupt latency!" << std::endl;
    }
    
    if (stats.pending_requests > 100) {  // 积压 > 100
        std::cerr << "Warning: Too many pending interrupts!" << std::endl;
    }
    
    std::this_thread::sleep_for(std::chrono::seconds(1));
}
```

## 🐛 常见问题

### Q1: 为什么我的中断没有被处理？

**A:** 检查以下几点：
1. VM 是否正确注册到 VmManager
2. 中断消息的 receiver 是否为 `MODULE_VM_MANAGER`
3. 工作线程是否启动（调用 `VmManager::start()`）

### Q2: 如何知道中断是否超时？

**A:** 检查 `InterruptResult::is_timeout` 字段：
```cpp
if (result.is_timeout) {
    std::cout << "Interrupt timed out!" << std::endl;
} else {
    std::cout << "Return value: " << result.return_value << std::endl;
}
```

### Q3: 如何优化中断延迟？

**A:** 
1. 使用合适的优先级（HIGH/REALTIME 用于紧急中断）
2. 合理设置超时时间（避免过长等待）
3. 监控 `avg_latency_ms` 指标
4. 减少不必要的中断请求

### Q4: 中断统计何时清零？

**A:** 统计是累积的，不会自动清零。重启 VmManager 会重置统计。

## 🎯 最佳实践

### ✅ 推荐做法

```cpp
// 1. 使用专用的 InterruptType
req.set_interrupt_type(InterruptType::KEYBOARD);

// 2. 根据设备类型设置合理的超时
req.timeout_ms = (device_type == FAST_DEVICE) ? 500 : 2000;

// 3. 记录中断延迟
auto stats = vm_manager.get_interrupt_stats();
if (stats.avg_latency_ms > threshold) {
    // 性能告警
}

// 4. 清理未使用的中断请求
vm_manager.unregister_vm(vm_id);  // 自动清理 pending 中断
```

### ❌ 不推荐做法

```cpp
// 1. 所有中断都用相同的超时
req.timeout_ms = 2000;  // 不够灵活

// 2. 忽略统计信息
// 从不检查 get_interrupt_stats()

// 3. 手动修改优先级（应该让系统自动设置）
req.priority = InterruptPriority::HIGH;  // 应该用 set_interrupt_type()
```

## 核心设计

### 1. 策略模式实现

**SchedulerBase.h** - 抽象基类
```cpp
class SchedulerBase {
public:
    virtual void start() = 0;
    virtual void stop() = 0;
    virtual void register_vm(uint64_t vm_id, std::weak_ptr<baseVM> vm) = 0;
    virtual void block_vm(uint64_t vm_id, int reason) = 0;
    virtual void wake_vm(uint64_t vm_id) = 0;
    // ... 其他接口
};
```

**QuotaScheduler.h** - 具体策略
```cpp
class QuotaScheduler : public SchedulerBase {
    // 使用 PBDS 实现优先队列
    typedef tree<ScheduleTask, null_type, less<ScheduleTask>, 
                 rb_tree_tag, tree_order_statistics_node_update> priority_queue_t;
};
```

### 2. SchedulerManager - 上下文类

**职责**：
- 持有 `std::shared_ptr<SchedulerBase>`
- 提供 `set_scheduler()` 动态切换算法
- 所有调度操作委托给当前策略

```cpp
void SchedulerManager::block_vm(uint64_t vm_id, int reason) {
    if (current_scheduler_) {
        current_scheduler_->block_vm(vm_id, reason);
    }
}
```

### 3. PBDS（Policy-Based Data Structures）

**什么是 PBDS？**
- GCC 的 C++ 库扩展
- 提供高性能平衡树容器
- O(log n) 插入/删除/查找性能

**为什么选择 PBDS？**
| 数据结构 | 插入 | 删除 | 查找最小值 | 适用场景 |
|---------|------|------|-----------|---------|
| `std::set` | O(log n) | O(log n) | O(1) | 通用 |
| `std::priority_queue` | O(1) | O(n) | O(1) | 仅需要最大值 |
| **PBDS tree** | **O(log n)** | **O(log n)** | **O(1)** | **需要排名/顺序统计** |

**PBDS 优势**：
1. **自动平衡**：红黑树实现，无需手动维护
2. **顺序统计**：支持 `order_of_key()` 查询排名
3. **动态更新**：可以在运行时调整元素优先级

## 代码对比

### std::set 版本（旧）
```cpp
std::set<ScheduleTask> ready_queue_;

void allocate_quotas() {
    for (const auto& task : ready_queue_) {
        // 遍历所有元素
    }
}
```

### PBDS 版本（新）
```cpp
typedef tree<ScheduleTask, null_type, less<ScheduleTask>, 
             rb_tree_tag, tree_order_statistics_node_update> priority_queue_t;

priority_queue_t ready_queue_;

void allocate_quotas() {
    // PBDS 提供更高效的遍历
    for (const auto& task : ready_queue_) {
        // O(log n) 性能
    }
}

// 额外功能：可以查询 VM 的排名
int rank = ready_queue_.order_of_key(target_task);
```

## 性能提升

| 操作 | std::set | PBDS | 提升 |
|------|----------|------|------|
| 插入 | O(log n) | O(log n) | 持平 |
| 删除 | O(log n) | O(log n) | 持平 |
| 查询排名 | O(n) | **O(1)** | **质变** |
| 前 K 个元素 | O(n) | **O(K)** | **显著** |

**实际意义**：
- 当 VM 数量 > 100 时，PBDS 性能优势明显
- 支持"只调度前 N 个高优 VM"的优化策略
- 为未来高级调度算法奠定基础

## 可扩展性

### 轻松添加新算法

**示例：添加 CFS 调度器**

1. 创建 `CfsScheduler.h/cpp`
2. 继承 `SchedulerBase`
3. 实现 Linux CFS 算法
4. 使用时调用：
```cpp
auto cfs = std::make_shared<CfsScheduler>();
SchedulerManager::instance().set_scheduler(cfs);
```

**示例：添加 EDF 调度器**
- 同样方式实现 `EdfScheduler`
- 无需修改 VmManager 或 SchedulerManager

### 动态切换算法

```cpp
// 启动时使用 PBDS
SchedulerManager::instance().start();

// 运行时切换到 CFS（如果实现了）
auto cfs = std::make_shared<CfsScheduler>();
SchedulerManager::instance().set_scheduler(cfs);

// 查询当前算法
std::cout << "Using: " << SchedulerManager::get_current_scheduler_name() << std::endl;
```
